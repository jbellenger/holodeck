## Project Overview

This is a GraphQL Conf 2025 presentation system that converts Blender animations into browser-playable slide decks. It consists of two main components:

1. **deckgen** - A Blender add-on that generates a manifest during render
2. **Browser player** - An HTML/JS viewer that plays back rendered frames with keyboard navigation

## Development Commands

```bash
make setup      # Create venv and install test dependencies
make test       # Run all unit tests
make test-one TEST=test_add_frame   # Run a specific test
make serve      # Start the presentation server on port 8000
make clean      # Remove venv and caches
```

## Architecture

### Blender Add-on (deckgen/)

```
deckgen/
├── __init__.py                    # Entry point with hot-reload support
├── blender_manifest.toml
├── core/
│   └── manifest_generator.py      # Pure Python logic (no bpy, fully testable)
└── handlers/
    └── render_handlers.py         # Thin bpy adapters
```

The add-on separates business logic (`core/`) from Blender-specific code (`handlers/`):
- `ManifestGenerator` class handles frame collection and manifest generation (testable without Blender)
- Render handlers are thin adapters that delegate to `ManifestGenerator`

The `__init__.py` includes hot-reload support—use "Reload Scripts" in Blender (F3) to pick up changes without restarting.

Blender timeline markers become "stops" in the presentation where playback pauses.

### Browser Player (index.html)

Keyboard controls:
- **Space**: Play/pause (auto-stops at markers)
- **Arrow Left**: Jump to previous marker
- **Arrow Right**: Jump to next marker

The player reads `manifest.json` to get frame paths, fps, and marker positions.

### manifest.json

Generated by the Blender add-on, contains:
- `fps`: Animation frame rate
- `markers`: Frame numbers where playback should pause (slide transitions)
- `frames`: Ordered list of paths to rendered PNG frames in `render/`

### File Structure

- `demo.blend` - Blender project file
- `render/` - PNG image sequence output from Blender
- `deckgen/` - Blender add-on source (see structure above)
- `tests/` - pytest unit tests for the add-on
- `deckgen-venv/` - Python virtual environment for testing

## Workflow

1. Create/edit animation in `demo.blend`
2. Add timeline markers at slide transition points
3. Render animation (the deckgen add-on auto-generates `manifest.json`)
4. Run `./run-server.sh` and open `http://localhost:8000` to present
